La Li Lu Le Lo End to End Encrypted Messaging

The Security

La Li Lu Le Lo is an end to end encrypted messaging service that keeps all messages hidden, not only from outside parties but the host server and anyone with access to the database.

The encryption scheme works as follows:

First, when you sign up, a public and private RSA key-pair are generated using the node-forge encryption package. The public key is saved to the database as is but your private key is itself encrypted using your password as the AES key and is then stored in the database. When you sign in after that, the private key is decrypted using your password and ready to use throughout the rest of the app.

Next, when a conversation is created, all messages are encrypted using a pseudorandomly generated AES key. This key is then encrypted using each participant in the conversation's public RSA key, so each user has a unique encrypted AES key.

Finally, when you send messages in a conversation, it decrypts your version of the AES key and then encrypts and decrypts all messages using that AES key. This way we can take advantage of the speed and convenience of a single AES key but still keep that key completely hidden.

The Function

In addition to being secure, La Li Lu Le Lo also has many of the features expected from a proprietary messaging service. Using the twemoji package, La Li Lu Le Lo has full support for emojis. It also allows for media, images and videos including the webm, mp4, jpg, png and gif formats, to be shared through the service.

Commands

La Li Lu Le Lo has several commands that allow you to embed media, take polls and generate random numbers for fun.

!img <image url> will allow you to embed external images.

!webm <webm url> will allow you to embed external webms.

!youtube <youtube url> will allow you to embed external youtube videos.

!poll <question>|<answer1>|<answer2>|<answer3>... will allow you to take a poll with any number of answers that anyone in the conversation can answer. The question and answer choices to this poll are also encrypted so only those in the conversation will ever know the contents.

!roll <maximum> will generate a number between 1 and the maximum number and send it to the chat.

The Message Behind The Messaging

I started this project because many of my friends and I primarily use Facebook messenger to communicate but I always disliked it because of the numerous privacy violations and potential free speech concerns but couldn't offer a LIBRE alternative. And thus the app was born. With the encryption in place and guarantees of transparency made possible by the GPL 2, I know this app can be a tool for freedom of speech and data privacy. I hope that people find it not only good in terms of security and ethics, but also in terms of quality and usability.
